# Золотые тест-кейсы: strategy-agent

Используются при обновлении промпта субагента для проверки, что качество не ухудшилось.

---

## Тест 1: Еженедельный брифинг

**Вход:** «Понедельничный брифинг.»

**Ожидаемый результат:**
- 5 блоков по формату: Фокус, Рекомендации (3-5), Упущенные возможности, Ранние сигналы, Вопрос
- Каждая рекомендация: действие + источник + последствие бездействия + уровень уверенности
- Блок «Эволюция системы» с данными из calibration_log (или «недостаточно данных»)
- Нет абстрактных советов — только привязанные к фактам

**Красные флаги:**
- Рекомендации без источников («стоит задуматься о...»)
- Отсутствие уровня уверенности [ВЫСОКАЯ]/[СРЕДНЯЯ]/[ГИПОТЕЗА]
- Дублирование работы operations-agent (пересказ статусов вместо стратегического слоя)
- Блок эволюции основан на «общих соображениях», а не на calibration_log

---

## Тест 2: Анализ развилки

**Вход:** «Развилка: нанимать ML-инженера в штат или привлекать на ГПХ? Помоги решить.»

**Ожидаемый результат:**
- Минимум 2 варианта в таблице: аргументы за/против, риски
- Каждый аргумент привязан к файлу или записи в памяти (бюджет, обязательства по гранту)
- Рекомендация с обоснованием
- Учтён контекст: статус Сколково, бюджет гранта, текущий состав команды

**Красные флаги:**
- Абстрактные «плюсы/минусы» без привязки к данным проекта
- Рекомендации, противоречащие друг другу
- Не учтены финансовые ограничения (бюджет гранта)
- Отсутствие уровня уверенности
