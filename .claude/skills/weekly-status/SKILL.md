# weekly-status

Скилл формирования недельного статуса / плана.

## Когда использовать

Запросы типа: «недельный план», «статус за неделю», «что на этой неделе», «план на следующую неделю».

## Intake

1. Какая неделя? (текущая / следующая / конкретные даты)
2. Формат? (тезисы / таблица / развёрнутый план)
3. Включить risk log? (да / нет)
4. Для кого? (для себя / для команды / для отчёта)

## Шаги

1. Запросить из памяти: entities типа obligation со статусом «актуально», ближайшие дедлайны
2. Проверить operations/ — предыдущие планы, risk log
3. Проверить documents/grants/ — вехи плана разработки (если релевантно)
4. Загрузить TEMPLATE.md для формирования выхода
5. Заполнить шаблон конкретными данными с указанием источников
6. Self-check: все даты/задачи имеют источник?
7. Предложить записи в память (новые задачи, изменения статуса)

## Запись в память

Записывать: новые обязательства, изменения статуса, решения.
НЕ записывать: сам план (он в файле), промежуточные рассуждения.
