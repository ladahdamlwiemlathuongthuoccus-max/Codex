# onboarding-brief

Быстрое введение нового члена команды в контекст проекта RedPeak.

## Когда использовать

- Новый человек присоединяется к команде
- Нужно подготовить вводный документ
- Фрилансер или подрядчик начинает работу

## Intake

1. Кто новый участник? (имя, роль)
2. Что будет делать? (разработка / ML / дизайн / аналитика / администрирование)
3. Какой уровень доступа? (вся информация / только своя зона)
4. Формат? (документ / тезисы / устный брифинг)

## Шаги

1. Загрузить из памяти: текущие приоритеты, команда, статус проекта
2. Загрузить из памяти: contacts (команда)
3. Собрать релевантную информацию по зоне нового участника
4. Сформировать брифинг по шаблону ниже
5. Self-check: нет ли конфиденциальной информации, которую не нужно раскрывать

## Шаблон онбординг-брифинга

### 1. Что такое RedPeak
[2-3 предложения: продукт, миссия, стадия]

### 2. Команда
| Имя | Роль | Зона ответственности |
|---|---|---|
| ... | ... | ... |

### 3. Что сейчас происходит
[Текущий этап, ближайшие цели, ключевые дедлайны]

### 4. Твоя зона
[Что конкретно будет делать новый участник, с кем взаимодействует]

### 5. Где что лежит
| Что | Где |
|---|---|
| Код | [репозиторий] |
| Документы | [путь/сервис] |
| Коммуникация | [мессенджер/канал] |
| Задачи | [трекер] |

### 6. Первые шаги
1. [Конкретное действие 1]
2. [Конкретное действие 2]
3. [Конкретное действие 3]

## Конфиденциальность

- НЕ включать финансовые детали (суммы контрактов, зарплаты) без явного указания
- НЕ включать персональные данные команды (паспорта, адреса)
- Уровень детализации — по указанию руководителя

## Запись в память

Записывать: новый контакт (имя, роль, дата выхода).
