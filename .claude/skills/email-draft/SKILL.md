# email-draft

Черновики писем с адаптацией тона под адресата.

## Когда использовать

- Любое письмо: для команды, партнёра, ФСИ, Сколково, АСКОН, инвестора
- Ответ на входящее письмо
- Первое обращение / cold outreach

## Intake

1. Кому? (имя, роль, компания)
2. О чём? (тема в одном предложении)
3. Цель письма? (информировать / попросить / ответить / договориться)
4. Тон? (или определю автоматически по адресату)
5. Есть ли входящее письмо, на которое отвечаем?

## Автоопределение тона

| Адресат | Тон | Правила |
|---|---|---|
| Команда | Неформальный | Короткие предложения. Без канцелярита. Как в мессенджере, чуть формальнее. Можно "ребят", "го", "короче". |
| Партнёр / АСКОН | Деловой, живой | Профессионально, но без "в рамках осуществления". Конкретика. |
| ФСИ / Сколково | Официальный | Формулировки по стандартам. Ссылки на пункты договора/заявки. Цифры. |
| Инвестор | Уверенный, чёткий | Факты и метрики. Без заискивания. Структура: результат → план → запрос. |
| Незнакомый контакт | Вежливый, краткий | Кто мы → зачем пишем → что нужно. Максимум 5 предложений. |

## Шаги

1. Определить адресата и тон
2. Загрузить из памяти: контакт, история взаимодействия, обязательства
3. Если ответ на письмо, проанализировать входящее
4. Написать черновик по правилам тона
5. Self-check:
   - Нет длинных тире (—)
   - Нет канцелярита (кроме официальных писем в ФСИ/Сколково)
   - Цель письма понятна с первого абзаца
   - Есть конкретный call to action (если нужен)
6. Предложить тему письма (subject line)

## Запреты

- НЕ отправлять письмо самостоятельно (только черновик)
- НЕ указывать email-адреса из памяти без подтверждения
- НЕ писать от имени руководителя без явного указания

## Запись в память

НЕ записывать текст письма. Записывать (после подтверждения): договорённости из переписки, новые контакты.
