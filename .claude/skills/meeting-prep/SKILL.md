# meeting-prep

Скилл подготовки к созвонам и встречам.

## Когда использовать

Запросы типа: «подготовь к созвону с...», «повестка встречи», «что обсудить с...».

## Intake

1. С кем встреча? (имя, роль, компания)
2. Цель встречи? (статус / решение / переговоры / знакомство)
3. Есть ли конкретные вопросы, которые нужно обсудить?
4. Формат? (повестка / список вопросов / развёрнутая подготовка)

## Шаги

### Фаза 1: Intake + память (dispatcher)
1. Провести intake-опрос (см. выше)
2. Запросить из памяти: entities, связанные с этим контактом / компанией

### Фаза 2: Сбор данных (ПАРАЛЛЕЛЬНО, если встреча с контрагентом по договору)
- **operations-agent:** «Покажи открытые задачи и статусы, связанные с [контакт/компания]. Формат: список задач со статусами и источниками.»
- **legal-agent:** «Покажи обязательства и ключевые даты по договорам с [контрагент]. Формат: таблица обязательств.»

Если встреча внутренняя или ознакомительная — Фаза 2 не нужна, достаточно памяти.

### Фаза 3: Сборка повестки (dispatcher)
3. Загрузить TEMPLATE.md для формирования повестки
4. Заполнить шаблон данными из Фаз 1-2 с указанием источников
5. Reflection: все пункты обоснованы?
6. Предложить записи в память после встречи (решения, договорённости)

## Запись в память

После встречи (по результатам руководителя): решения, договорённости, следующие шаги.
