# Verification Policy

## Обязательные проверки

- Проверяй результат по текущим файлам проекта, а не по памяти.
- Для внешних изменчивых фактов используй актуальные официальные источники.
- Для кода запускай релевантные тесты или минимальный smoke.
- Для документов проверяй структуру и наличие созданного файла.
- Для MCP проверяй JSON/TOML, существование путей и доступность зависимостей.

## Запрещено

- Выдавать неподтверждённые факты как проверенные.
- Ссылаться на `D:\REDPEAK\Agent systems\AgentHQ` как активный workspace.
- Выполнять чувствительное браузерное действие без финального подтверждения.
- Массово переписывать память без подтверждения руководителя.
