# /fsi-report

Сборка отчёта по гранту ФСИ «Старт-ЦТ-1». Критичная задача — документ уходит третьим лицам.

## Intake перед запуском

Уточнить у руководителя (если не указано):
1. Промежуточный или итоговый?
2. За какой период (дата начала и конца этапа)?
3. Какой номер этапа по плану?
4. Есть ли нестандартные отклонения от плана, о которых нужно упомянуть?

## Параллельные задачи (accounting ‖ operations)

1. **accounting-agent** — расходы по смете гранта за период. Привязать каждую статью к первичке (`documents/finance/`). Сверить фактические расходы с утверждённой сметой.

2. **operations-agent** — статус вех разработки за период. Привязать к плану разработки. Указать: что сделано, что в процессе, какие отклонения.

ПАРАЛЛЕЛЬНО — оба субагента независимы.

## Последовательно (сборка и оформление)

3. Вызвать `fsi-report` skill (если есть) — либо диспетчер собирает по стандарту ФСИ: титульный лист, цели этапа, выполненные работы, расходы по смете, план на следующий этап, подписи.

4. **Evaluator-agent** — обязательный двойной ревью (критическая задача, уходит в ФСИ).

5. **Codex documents** — собрать .docx файл с форматированием под стандарт отчётности ФСИ.

## Формат результата

- Основной: `.docx` в `documents/grants/отчёт_фси_ЭтапN_YYYY-MM-DD.docx`
- Черновик: `documents/grants/отчёт_фси_ЭтапN_YYYY-MM-DD.md`

## Верификация перед выдачей

- Каждая сумма в расходах имеет путь к первичке (счёт, акт, платёжка)
- Каждая веха имеет конкретное доказательство (коммит, демо, акт)
- Сумма расходов = сумма по позициям (арифметика)
- Плановые vs фактические расходы сверены
- Evaluator выдал PASS
- Руководитель подтвердил перед сохранением финального .docx
