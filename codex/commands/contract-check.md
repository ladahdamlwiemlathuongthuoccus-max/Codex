# /contract-check

Проверка договора с одновременной сверкой финансовой части с бюджетом проекта.

## Intake

Уточнить:
1. Какой файл договора анализируем? (точный путь)
2. Что конкретно проверить: полный разбор / конкретный пункт / сравнение с редакцией / соответствие смете?
3. С чем сверять бюджет: общий бюджет проекта / смета гранта / утверждённый план?

## Параллельные задачи (legal ‖ accounting)

1. **legal-agent** — юридический анализ договора:
   - Обязательства сторон (таблица)
   - Сроки и дедлайны
   - Ответственность, неустойки
   - IP и конфиденциальность
   - Правовые риски
   - Формат: таблица обязательств + отдельный блок рисков

2. **accounting-agent** — финансовая сверка:
   - Сумма договора
   - Этапы платежей
   - Соответствие смете гранта или бюджету проекта
   - Влияние на burn rate
   - Первичка для каждого оплаченного этапа

ПАРАЛЛЕЛЬНО.

## Последовательно (сводка)

3. Диспетчер собирает результаты в единый документ:
   - Юридические обязательства
   - Финансовые обязательства
   - Сверка с бюджетом/сметой: соответствует / требует правок / превышение
   - Общие риски (правовые + финансовые)
   - Рекомендация: подписывать / доработать / отклонить

## Evaluator (обязательный)

Критическая задача — документ может быть подписан. Вызов `evaluator-agent` обязателен.

## Верификация

- Каждый пункт договора имеет ссылку: `путь_к_файлу, п. N.N`
- Каждая сумма сверена с первичкой (для уже оплаченных этапов)
- Соответствие смете проверено по конкретному файлу сметы
- Нет придуманных пунктов (сверить номера с реальным документом)
- Evaluator: PASS
