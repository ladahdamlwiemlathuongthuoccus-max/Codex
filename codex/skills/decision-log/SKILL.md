# decision-log

Фиксация решений с контекстом и обоснованием.

## Когда использовать

- Принято важное решение, которое нужно задокументировать
- Нужно вспомнить, почему было принято решение
- Обзор решений за период
- Переоценка ранее принятого решения

## Intake

### Для записи нового решения
1. Какое решение? (что решили)
2. Почему? (контекст и обоснование)
3. Какие альтернативы рассматривались?
4. Кто принял решение?
5. Есть ли срок пересмотра?

### Для обзора
1. По какой теме? (или за какой период)

## Шаблон записи

### DEC-[NNN]: [Название решения]

| Поле | Значение |
|---|---|
| Дата | [YYYY-MM-DD] |
| Принял | [имя] |
| Статус | действует / пересмотрено / отменено |
| Область | продукт / технология / бизнес / команда / юр. / финансы |

#### Контекст
[Что происходило, почему встал вопрос — 2-4 предложения]

#### Решение
[Что именно решили — чётко и однозначно]

#### Обоснование
[Почему выбран этот вариант — факты и логика]

#### Альтернативы
| Вариант | Почему отклонён |
|---|---|
| [Вариант B] | [причина] |
| [Вариант C] | [причина] |

#### Последствия
[Что меняется в результате этого решения]

#### Пересмотр
[Когда / при каких условиях решение может быть пересмотрено]

## Файл хранения

Основной файл: `operations/decision_log.md`
Если файла нет — создать при первом добавлении.

## Шаги

### Запись нового решения
1. Загрузить из памяти: entities типа decision (проверить дубликаты и противоречия)
2. Загрузить operations/decision_log.md (если есть)
3. Сформировать карточку по шаблону
4. Показать руководителю для подтверждения
5. Добавить в файл и предложить запись в память

### Обзор решений
1. Загрузить из памяти: entities типа decision по запрошенной теме
2. Загрузить operations/decision_log.md
3. Отфильтровать и показать

## Запись в память

Записывать: решение (суть + обоснование + дата). Тип entity: decision.
НЕ записывать: отклонённые альтернативы (они в файле).
