# risk-log

Управление рисками проекта RedPeak.

## Когда использовать

- Добавление нового риска
- Обзор текущих рисков
- Переоценка рисков (изменились обстоятельства)
- Подготовка раздела "риски" для отчёта или встречи

## Intake

1. Что делаем? (добавить риск / обзор / переоценка / для отчёта)
2. Если новый риск: опишите ситуацию
3. Контекст: к какой части проекта относится? (грант / продукт / команда / финансы / юр.)

## Шаги

### Добавление нового риска
1. Загрузить из памяти: entities типа risk
2. Проверить: нет ли дубликата
3. Оценить вероятность (1-5) и импакт (1-5)
4. Определить владельца риска (кто мониторит)
5. Предложить план митигации
6. Сформировать карточку и предложить запись в память

### Обзор рисков
1. Загрузить из памяти: все entities типа risk со статусом "актуально"
2. Загрузить operations/risk_log.md (если есть)
3. Сформировать таблицу, отсортированную по приоритету (вероятность x импакт)
4. Отметить риски, которые могли измениться с последнего обзора

## Формат карточки риска

| Поле | Значение |
|---|---|
| ID | RISK-[NNN] |
| Описание | [Что может произойти] |
| Категория | грант / продукт / команда / финансы / юр. / внешний |
| Вероятность | 1 (низкая) — 5 (почти точно) |
| Импакт | 1 (минимальный) — 5 (критический) |
| Приоритет | вероятность x импакт |
| Владелец | [Кто мониторит] |
| Митигация | [Что делаем, чтобы снизить] |
| Триггер | [По какому сигналу риск реализуется] |
| Статус | открыт / мониторинг / реализовался / закрыт |

## Формат реестра (таблица)

| ID | Риск | Кат. | P | I | Приоритет | Владелец | Митигация | Статус |
|---|---|---|---|---|---|---|---|---|
| RISK-001 | ... | ... | 3 | 4 | 12 | ... | ... | открыт |

## Файл хранения

Основной файл: `operations/risk_log.md`
Если файла нет — создать при первом добавлении.

## Запись в память

Записывать: новые риски, изменения статуса рисков.
НЕ записывать: промежуточные оценки, закрытые риски (они в файле).
