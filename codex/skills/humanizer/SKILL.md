# Humanizer — превращение текста в живой человеческий язык

## Цель

Переписать входной текст так, чтобы он был неотличим от написанного живым человеком. Никакой робот, никакая нейросеть — только естественный русский язык.

## Процесс

1. Получи текст от пользователя
2. Определи контекст: для кого текст (команда, партнёр, руководство)
3. Если контекст не указан — по умолчанию это **сообщение для команды**
4. Перепиши по правилам ниже
5. Выдай результат. Без преамбулы "Вот переписанный текст:" — просто текст

## Правила переписывания

### ЗАПРЕЩЕНО (признаки нейросети — убирать безжалостно)

- **Длинное тире (—)** — заменять на точку, запятую или перестроение предложения
- **Канцелярит**: "в рамках", "осуществить", "обеспечить реализацию", "произвести оплату", "в целях оптимизации", "данный", "является", "представляет собой"
- **Вводные-паразиты**: "Важно отметить, что...", "Следует подчеркнуть...", "Необходимо учитывать...", "Стоит отметить...", "Хочется подчеркнуть..."
- **Однотипная структура**: все абзацы по схеме "тезис → объяснение → вывод"
- **Комплименты задаче**: "Отличный вопрос!", "Это важная тема!"
- **Списки-пятёрки**: списки из ровно 5 пунктов, где каждый начинается одинаково
- **Избыточные наречия**: "действительно", "безусловно", "несомненно", "определённо"
- **Пассивный залог** без необходимости: "было принято решение" → "мы решили"
- **"Кроме того" / "Более того" / "Помимо этого"** — просто начни новое предложение
- **Парные конструкции**: "не только X, но и Y" — часто можно проще
- **"В контексте" / "В данном контексте"** — убрать
- **Абстрактные существительные**: "осуществление процесса" → конкретный глагол

### ОБЯЗАТЕЛЬНО

- **Короткие предложения**. Средняя длина 8-15 слов. Длинные — разбивать
- **Активный залог**. "Мы сделали" вместо "было сделано"
- **Конкретные глаголы**. "Отправил", "починил", "договорился" — не "обеспечил", "реализовал"
- **Разная длина предложений**. Чередовать короткие и средние. Иногда одно слово. Да
- **Живые связки**. "Короче", "в общем", "по итогу", "ещё момент" — если тон позволяет
- **Прямой порядок**: подлежащее → сказуемое → дополнение. Инверсию — только для акцента
- **Местоимения**: "мы", "я", "нам" вместо безличных конструкций
- **Конкретика вместо абстракций**: "за 2 дня" вместо "в кратчайшие сроки"

### Адаптация по контексту

| Контекст | Тон | Особенности |
|---|---|---|
| Команда (по умолчанию) | Неформальный, дружеский | Можно "короче", "го", "ребят". Без "уважаемые коллеги" |
| Партнёр / контрагент | Деловой, но живой | Без канцелярита, но и без сленга. Конкретно и по делу |
| Руководство / инвестор | Чёткий, уверенный | Факты, цифры, минимум воды. Без заискивания |

## Примеры трансформации

### Было (нейросеть):
"Важно отметить, что в рамках реализации проекта были достигнуты значительные результаты — команда успешно осуществила интеграцию модуля, что позволило обеспечить существенное повышение производительности системы."

### Стало (человек):
"Интеграцию модуля закончили. Производительность выросла, точные цифры скину завтра."

### Было (нейросеть):
"Необходимо учитывать, что в текущих условиях представляется целесообразным произвести корректировку сроков выполнения задач, поскольку ряд факторов оказывает существенное влияние на ход работ."

### Стало (человек):
"Сроки сдвигаются. Причина: [конкретная причина]. Новый дедлайн: [дата]."

## Финальная проверка

Перед выдачей результата проверь:
- [ ] Нет ни одного длинного тире (—)
- [ ] Нет слов: "данный", "является", "в рамках", "осуществить", "обеспечить", "реализовать", "представляет собой"
- [ ] Нет вводных-паразитов
- [ ] Средняя длина предложения < 15 слов
- [ ] Предложения разной длины (не все одинаковые)
- [ ] Текст звучит как сообщение живого человека, а не пресс-релиз

Если хоть один пункт не выполнен — переписать ещё раз.
