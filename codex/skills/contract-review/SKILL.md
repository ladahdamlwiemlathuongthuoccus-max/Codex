# contract-review

Скилл анализа договоров и юридических документов.

## Когда использовать

Запросы типа: «разбери договор», «таблица обязательств», «анализ рисков по договору», «сравни две редакции», «что мы обязаны по договору с...».

## Intake

1. Какой документ? (путь к файлу в documents/contracts/)
2. Что нужно: полный разбор / таблица обязательств / анализ рисков / конкретный пункт?
3. Есть ли встречный документ для сравнения?
4. Формат результата? (таблица / записка / чек-лист)

## Шаги

1. Открыть конкретный файл договора
2. Запросить из памяти: entities типа obligation, связанные с этим контрагентом
3. Провести анализ — загрузить CHECKLIST.md для проверки обязательных пунктов
4. Если нужен анализ рисков — загрузить RISKS.md для типовых рисков
5. Каждый упомянутый пункт — сверить с документом (номер пункта существует?)
6. Сформировать результат с указанием источников: [файл, пункт N.N]
7. Self-check
8. Предложить записи в память (обязательства, риски, ключевые даты)

## Контракт данных (обязательный формат выхода)

### Обязательства сторон
| № п/п | Пункт договора | Обязанная сторона | Суть обязательства | Срок | Последствия нарушения |
|---|---|---|---|---|---|
| 1 | п. N.N | ... | ... | ... | ... |

### Выявленные риски
| Риск | Пункт договора | Вероятность | Рекомендация |
|---|---|---|---|
| ... | п. N.N | высокая/средняя/низкая | ... |

### Ключевые даты
| Дата | Событие | Пункт договора |
|---|---|---|
| ... | ... | п. N.N |

**Источник:** [полный путь к файлу договора]

## Запись в память

Записывать: обязательства (кто, что, когда, последствия), ключевые даты, выявленные риски.
НЕ записывать: текст договора целиком, промежуточный анализ.
