# task-card

Создание структурированной карточки задачи.

## Когда использовать

- Нужно поставить задачу разработчику или члену команды
- Формализовать расплывчатый запрос в конкретную задачу
- Подготовить описание для трекера задач

## Intake

1. Что нужно сделать? (суть задачи)
2. Для кого задача? (исполнитель)
3. К какому сроку?
4. Контекст: почему эта задача нужна?
5. Есть ли зависимости от других задач?

## Шаблон карточки

### [Название задачи]

**Исполнитель:** [имя]
**Дедлайн:** [дата]
**Приоритет:** P0 (блокер) / P1 (важно) / P2 (желательно) / P3 (бэклог)

#### Контекст
[1-3 предложения: почему эта задача нужна, что она решает]

#### Что сделать
1. [Конкретный шаг 1]
2. [Конкретный шаг 2]
3. [Конкретный шаг 3]

#### Acceptance Criteria
- [ ] [Критерий 1 — проверяемое условие]
- [ ] [Критерий 2 — проверяемое условие]
- [ ] [Критерий 3 — проверяемое условие]

#### Зависимости
- [Задача X должна быть завершена до начала]
- или: нет зависимостей

#### Результат
[Что должно быть готово: файл, функция, документ, деплой]

## Правила

- Acceptance criteria должны быть проверяемыми (да/нет), а не расплывчатыми
- "Что сделать" — конкретные действия, а не абстракции
- Если задача слишком большая (> 5 шагов), предложить разбить на подзадачи
- Тон: прямой, конкретный, без воды

## Запись в память

НЕ записывать карточки задач (они в трекере или файле).
Записывать: если задача создаёт новое обязательство (дедлайн, зависимость).
