# fsi-report

Скилл подготовки отчётов по гранту ФСИ «Старт-ЦТ-1».

## Когда использовать

Запросы типа: «отчёт ФСИ», «отчёт по гранту», «комплектность документов ФСИ», «что нужно для отчёта».

## Intake

1. Какой отчёт? (промежуточный / итоговый / конкретный этап)
2. За какой период?
3. Что нужно: полный отчёт / проверка комплектности / черновик раздела?
4. Формат? (docx / чек-лист / таблица)

## Шаги

### Фаза 1: Intake + загрузка (dispatcher)
1. Провести intake-опрос (см. выше)
2. Загрузить REFERENCE.md — требования ФСИ к отчёту
3. Запросить из памяти: entities типа obligation, связанные с ФСИ

### Фаза 2: Сбор данных (ПАРАЛЛЕЛЬНО)
Запустить два субагента одновременно:
- **accounting-agent:** «Собери расходы за [период] по статьям сметы гранта ФСИ. Формат: см. контракт данных ниже.»
- **operations-agent:** «Собери статус выполнения вех плана разработки за [период]. Формат: см. контракт данных ниже.»

### Фаза 3: Сборка (dispatcher)
5. Если проверка комплектности — загрузить CHECKLIST.md
6. Если формирование отчёта — загрузить TEMPLATE.md
7. Заполнить шаблон данными из Фазы 2 с указанием конкретных источников
8. Reflection (перечитать как ревьюер, проверить полноту)
9. Предложить записи в память (статус отчёта, выявленные проблемы)

## Контракты данных между фазами

### accounting-agent → dispatcher (Фаза 2 → 3)
Обязательный формат выхода:

| Статья сметы | План (₽) | Факт (₽) | Отклонение (₽) | Документ-основание |
|---|---|---|---|---|
| ФОТ (ГПХ) | ... | ... | ... | finance/[файл] |
| Взносы | ... | ... | ... | finance/[файл] |
| ... | ... | ... | ... | ... |

**Итого:** план ₽ / факт ₽ / отклонение ₽
**Примечания:** [если есть проблемы с документами или расхождения]

### operations-agent → dispatcher (Фаза 2 → 3)
Обязательный формат выхода:

| № вехи | Название | Плановый срок | Фактический статус | Результат | Источник |
|---|---|---|---|---|---|
| 1 | ... | ... | выполнено / в работе / не начато | ... | documents/grants/[файл] |
| ... | ... | ... | ... | ... | ... |

**Резюме:** [N] из [M] вех выполнено. Отставание: [да/нет, по каким вехам]

## Запись в память

Записывать: статус отчёта, выявленные недостачи, дедлайн подачи.
